# Чеклист требований ВКР

Соответствие функциональным и нефункциональным требованиям (раздел 1.3).

## Функциональные требования

| № | Требование | Статус | Примечание |
|---|------------|--------|------------|
| 1 | Панель админа: добавление, редактирование, удаление книг | ✅ | AdminBookForm, AdminBookList |
| 2 | Мониторинг резервирований и обращений | ✅ | AdminRequestsPage, выдачи в Borrowing |
| 3 | Поиск по ключевым атрибутам (название, автор) | ✅ | getBooks: $or по title и author |
| 4 | Фильтрация по жанру | ✅ | CatalogSearch, genre в API |
| 5 | Бронирование ресурсов | ✅ | borrowBook, returnBook |
| 6 | Получение материалов в PDF/TXT | ✅ | uploadBookFile, readBookFile |
| 7 | Настройки чтения (цвет, шрифт, масштаб) | ✅ | ReadBookPage |
| 8 | Модуль обращений (создание, отслеживание) | ✅ | MyRequestsPage, AdminRequestsPage |
| 9 | Разграничение ролей (user / admin) | ✅ | JWT, AdminRoute, проверка на сервере |
| 10 | Валидация входных данных | ✅ | validation.js, authController, сервер |
| 11 | Транзакционная согласованность выдачи | ✅ | Borrowing.create + Book.available последовательно |
| 12 | Модульность сервера (маршруты, контроллеры, модели) | ✅ | routes/, controllers/, models/ |

| 1 | Безопасность данных, защита от НСД | ✅ JWT, bcrypt |
| 2 | Устойчивость при одновременных обращениях | ✅ Mongoose, индексы |
| 3 | Целостность БД | ✅ Схемы Mongoose |
| 4 | Совместимость с браузерами (ПК, мобильные) | ✅ React, адаптивные стили |
| 5 | Расширяемость архитектуры | ✅ Модульная структура |
