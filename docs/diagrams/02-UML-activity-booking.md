# UML: Диаграмма активности — сценарий бронирования книги

Описывает поток операций при выдаче книги пользователю. Соответствует разделу 1.4 ВКР.

## Диаграмма активности

```mermaid
flowchart TD
    Start([Начало]) --> A[Пользователь нажимает «Взять»]
    A --> B{Авторизован?}
    B -->|Нет| C[Редирект на /login]
    B -->|Да| D[POST /api/books/:id/borrow]
    D --> E[Проверка JWT]
    E --> F{Токен валиден?}
    F -->|Нет| G[401 Unauthorized]
    F -->|Да| H[Поиск книги по ID]
    H --> I{Книга найдена?}
    I -->|Нет| J[404 Книга не найдена]
    I -->|Да| K{Книга доступна?}
    K -->|Нет| L[400 Книга уже выдана]
    K -->|Да| M[Создание записи Borrowing]
    M --> N[Обновление Book.available = false]
    N --> O[Сохранение в БД]
    O --> P[200 Книга успешно выдана]
    P --> Q[Обновление UI каталога]
    Q --> End([Конец])
    C --> End
    G --> End
    J --> End
    L --> End
```

## Упрощённая диаграмма (основной поток)

```mermaid
flowchart LR
    A[Запрос выдачи] --> B{Проверки}
    B -->|OK| C[Создать Borrowing]
    C --> D[Book.available = false]
    D --> E[Ответ успех]
    B -->|Ошибка| F[Сообщение об ошибке]
```

## Описание этапов

1. **Инициация** — пользователь нажимает кнопку «Взять» в каталоге.
2. **Проверка авторизации** — наличие JWT-токена.
3. **Проверка доступности** — книга существует и `available === true`.
4. **Транзакционная операция** — создание Borrowing и обновление Book в рамках логической последовательности.
5. **Ответ клиенту** — обновление интерфейса (каталог, «Мои книги»).
