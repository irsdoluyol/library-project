# План тестирования и тестовые сценарии

Документ описывает план тестирования веб-приложения онлайн-библиотеки в соответствии с разделом 3.3 ВКР.

## 1. Цели тестирования

- Проверка корректности работы ключевых функций системы
- Подтверждение соответствия функциональным и нефункциональным требованиям
- Выявление дефектов до развёртывания

## 2. Уровни тестирования

### 2.1 Unit-тесты (модульные)

Проверка отдельных функций и компонентов в изоляции.

| Модуль | Сценарий | Ожидаемый результат |
|--------|----------|---------------------|
| `validateRegister` | Валидные данные (имя, email, пароль) | Пустой объект ошибок |
| `validateRegister` | Имя короче 2 символов | Ошибка по полю name |
| `validateRegister` | Некорректный email | Ошибка по полю email |
| `validateRegister` | Пароль короче 6 символов | Ошибка по полю password |
| `validateRegister` | Фамилия необязательна | При пустой фамилии — без ошибок |
| `BookCard` | Рендер карточки книги | Отображаются название и автор |
| `BookCard` | Кнопка «Взять» при авторизации | Вызов onBorrow при клике |
| `BookCard` | Книга недоступна | Текст «Недоступно» |

### 2.2 Интеграционные сценарии (ручная проверка / E2E)

Проверка взаимодействия модулей через API и UI.

| Сценарий | Шаги | Ожидаемый результат |
|----------|------|---------------------|
| Регистрация | Заполнить форму, отправить | Создан пользователь, редирект на каталог |
| Вход | Ввести email/пароль | Получен JWT, отображён интерфейс пользователя |
| Поиск в каталоге | Ввести текст, выбрать жанр | Отфильтрованный список книг |
| Бронирование | Нажать «Взять» у доступной книги | Книга выдана, статус «Недоступно» |
| Возврат | Нажать «Вернуть» в «Мои книги» | Книга возвращена, исчезла из списка |
| Чтение | Нажать «Читать» у выданной книги | Открыт читатель с настройками |
| Обращение | Создать обращение | Запись в БД, отображение в админке |
| Админ: добавление книги | Заполнить форму, загрузить файл | Книга в каталоге, файл доступен |

### 2.3 Проверка безопасности

| Угроза | Мера защиты | Проверка |
|--------|-------------|----------|
| XSS | Экранирование в React, Content-Security-Policy | Ввод `<script>` в поля — не выполняется |
| SQL-инъекция | Mongoose, параметризованные запросы | Спецсимволы в поиске не ломают запрос |
| НСД | JWT, проверка роли на сервере | Запросы без токена / с ролью user к /admin — 401/403 |
| Утечка паролей | bcrypt, пароли не в ответах API | В ответах нет поля password |

## 3. Инструменты

- **Vitest** — unit-тесты (клиент и сервер)
- **@testing-library/react** — тестирование React-компонентов
- **Supertest** — API-тесты сервера
- **mongodb-memory-server** — in-memory MongoDB для API-тестов
- Ручная проверка — интеграционные и E2E-сценарии

## 4. Запуск тестов

**Клиент:**
```bash
cd client
npm install
npm test
```

**Сервер:**
```bash
cd server
npm install
npm test
```

При первом запуске серверных тестов MongoMemoryServer скачивает бинарник MongoDB (~27 МБ) в `node_modules/.cache/mongodb-binaries`. Последующие запуски используют кэш.

Режим watch (при изменении файлов):

```bash
# клиент
npm run test:watch

# сервер
cd server && npx vitest
```

## 5. Покрытие

**Клиент (unit):**
- Валидация регистрации (`validation.js`)
- Компонент `BookCard`, `BorrowingItem`, `CatalogSearch`
- API-функции (`booksApi`)

**Сервер:**
- **Unit:** `escapeRegex`, `validateEnv`
- **API:** health, auth (register/login), books (список, поиск, пагинация), admin (CRUD книг), borrow/return, requests (создание, мои обращения, admin: все + обновление статуса)
